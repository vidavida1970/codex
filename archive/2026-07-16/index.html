오늘의 학습목표

업무의 최신성·자료량·복잡도·최종 산출물에 따라 맞는 ChatGPT 작업 방식을 30초 안에 선택합니다.

핵심 개념 1분

도구는 강한 순서가 아니라 업무에 맞는 순서로 고릅니다. 작은 일에 큰 조사를 쓰면 느리고, 최신 근거가 필요한 일에 일반 대화만 쓰면 위험합니다.

일반 대화

문구 초안·요약·아이디어처럼 최신 외부 근거가 필요 없는 짧은 작업

Search

정책·행사·뉴스처럼 현재 정보와 바로 확인할 링크가 필요한 작업

Deep Research

여러 출처를 비교하고 근거가 남는 조사 보고서가 필요한 복합 과제

파일 분석

RFP·보도자료·CSV·이미지처럼 내가 가진 자료가 판단의 중심인 작업

Projects

같은 기관·브랜드·제안 업무의 지침과 자료를 여러 대화에서 이어갈 때

Work

조사부터 문서·슬라이드·사이트까지 여러 단계를 실제 결과물로 완성할 때

선택 공식: 최신 정보? → Search ｜ 여러 출처 비교? → Deep Research ｜ 내 자료 중심? → 파일 분석 ｜ 반복 맥락? → Projects ｜ 완성물까지 장기 실행? → Work

잘못된 사용법 vs 개선 예시

BAD

2027년 공공기관 홍보 트렌드와 경쟁기관 사례를 조사해 제안서 30장을 만들어줘.

검색·출처 검증·파일 분석·슬라이드 제작을 한 문장에 섞어 작업 방식과 완료 기준이 흐려집니다.

BETTER

먼저 조사 방식과 산출물 단계를 선택하게 합니다.

Search로 빠른 스캔 → Deep Research로 핵심 근거 비교 → RFP 파일 분석 → Work로 제안서 초안 제작처럼 연결합니다.

개선된 지시

이 업무를 ①최신 사실 확인 ②출처 비교 조사 ③첨부 RFP 요구사항 분석 ④30장 제안서 제작으로 나누라. 각 단계에 가장 적합한 ChatGPT 작업 방식을 추천하고, 먼저 1단계의 조사 질문·공식 출처 우선순위·완료 기준만 제시하라.

국내 사례 확인

검증 가능한 ‘작업 방식 선택’ 성과 사례 미확인

국내 기업의 생성형 AI 도입 보도는 확인됐지만, 일반 대화·Search·Deep Research·파일 분석을 업무별로 나눠 적용한 과정과 측정 결과를 함께 공개한 원문은 확인하지 못했습니다. 따라서 임의의 사례나 성과 수치를 만들지 않았습니다.

애드아이티 적용 포인트
공공기관 제안 업무를 ‘RFP 파일 분석 → 정책·경쟁기관 Search → 근거 비교 Deep Research → 장기 제작 Work’로 분리하면, 조사 누락과 출처 없는 문장을 줄이고 단계별 검수가 쉬워집니다.

2분 따라 하기

  1. 오늘 할 일 하나를 적습니다. 예: “청년정책 캠페인 제안서 초안 만들기”.
  2. 최신 정보가 필요한가, 여러 출처를 비교하는가, 첨부파일이 핵심인가, 여러 단계의 완성물이 필요한가에 예/아니요로 답합니다.
  3. 아래 프롬프트를 실행해 추천 작업 방식과 첫 단계만 확정합니다.

복사형 실전 프롬프트

다음 업무를 시작하기 전에 가장 적합한 ChatGPT 작업 방식을 선택하라.
[업무] {예: 공공기관 청년정책 캠페인 제안서 30장 초안 제작}
[보유 자료] {RFP, 기관 소개서, 기존 제안서, 데이터 등}
[최종 결과물] {문서·표·슬라이드·사이트 등}
[판단 질문] 1. 실행일 기준 최신 외부 정보가 필요한가? 2. 여러 출처를 비교하고 인용해야 하는가? 3. 첨부한 파일·데이터가 판단의 중심인가? 4. 같은 기관 맥락을 여러 대화에서 반복 사용할 것인가? 5. 조사부터 제작까지 여러 단계를 장시간 수행해야 하는가?
[선택 후보] 일반 대화, Search, Deep Research, 파일 분석, Projects, Work
[출력 형식] - 추천 작업 방식 1순위와 이유 - 함께 사용할 보조 방식 - 사용하지 않아도 되는 방식과 이유 - 첫 실행 단계 3개 - 첫 단계의 완료 기준 3개
확인되지 않은 기능명은 만들지 말고, 현재 ChatGPT 화면에서 실제 선택 가능한 제품 용어만 사용하라. 보유 자료에 개인정보·비공개 정보가 있으면 입력 전 보안 검토가 필요하다고 경고하라.

결과 체크 3문항

  • ✓ 최신성·출처·파일·반복 맥락·실행 복잡도를 기준으로 골랐는가?
  • ✓ 왜 이 방식을 쓰고 다른 방식은 쓰지 않는지 설명되는가?
  • ✓ 첫 단계의 산출물과 완료 기준이 한눈에 보이는가?

자주 하는 실수

  • 어려운 업무일수록 무조건 Deep Research를 선택하기
  • 최신 정책이나 통계를 일반 대화의 기억만으로 작성하기
  • RFP가 있는데 파일 분석 없이 추측시키기
  • Projects를 자동 기억 공간으로 오해하기
  • 비공개 제안서·개인정보를 보안 정책 확인 없이 업로드하기
