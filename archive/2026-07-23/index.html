5-MINUTE WORK CLASS · 2026.07.23

큰 업무를 한 번에 맡기면 오류도 한 번에 커집니다. 단계마다 작은 완료를 만드세요.

오늘의 학습목표
복잡한 업무를 단계별 산출물과 검토 지점으로 나눠, 앞 단계의 오류가 최종 결과까지 번지는 것을 막는다.

핵심 개념 1분

작업 분해는 긴 프롬프트를 여러 문장으로 나누는 일이 아닙니다. 각 단계에 입력·작업·산출물·통과 기준을 붙이는 업무 설계입니다.

1정의목표·사용자·최종 완료 기준을 확정
2분석자료를 읽고 요구사항·사실·빈칸을 분리
3설계전략·구성·우선순위를 결정
4제작승인된 설계를 기준으로 초안을 작성
5검수근거·누락·형식·리스크를 점검해 수정

핵심은 각 단계가 다음 단계의 검증된 입력이 되도록 만드는 것입니다. 전략을 승인하기 전에 문구부터 쓰면, 잘못된 방향을 예쁘게 완성하게 됩니다.

애드아이티 업무 예시

공공기관 홍보 제안은 RFP 요구사항표 → 평가항목별 과제 → 타깃 인사이트 → 채널 전략 → 콘텐츠 구성 → 장표 초안 → 심사표 검수로 나눕니다. 각 단계가 끝날 때 “확인된 사실·추론·추가 자료”를 분리해 승인한 뒤 다음 단계로 넘어갑니다.

단계 분해 전후를 측정한 공개 사례 미확인

국내 기업·공공기관 자료에서 동일 업무를 “한 번에 요청”과 “단계별 요청”으로 비교한 검증 가능한 성과 자료는 확인하지 못했습니다. 공개되지 않은 수치를 만들지 않고 오늘의 미니 실습으로 직접 측정합니다.

내 업무 적용 포인트

지금 진행 중인 제안서에서 가장 되돌리기 비싼 지점 하나를 찾고, 그 직전에 사람 승인 게이트를 넣으세요.

2분 따라 하기

  1. 복잡한 업무 하나와 최종 결과물을 한 줄로 적습니다.
  2. 시작부터 완료까지 동사 5개로 나눕니다.
  3. 각 단계의 산출물과 통과 기준을 한 줄씩 붙입니다.
  4. 되돌리기 비용이 큰 단계 앞에 ‘승인 후 진행’을 표시합니다.

복사형 실전 프롬프트

[최종 목표] {완성할 결과물과 사용처} [제공 자료] {RFP·사업자료·기존 콘텐츠·통계 등} [완료 기준] {누가 무엇을 확인하면 완료인지} 이 업무를 4~7개의 실행 단계로 분해하라. 각 단계는 아래 필드를 포함한다. | 단계 | 입력 자료 | 수행 작업 | 단계 산출물 | 통과 기준 | 사람 확인 | [분해 원칙] - 앞 단계 산출물이 다음 단계 입력이 되게 할 것 - 조사·판단·작성·검수를 한 단계에 섞지 말 것 - 근거가 부족하면 ‘추가 자료 필요’로 표시할 것 - 되돌리기 비용이 큰 결정 앞에는 승인 게이트를 둘 것 - 단계별 예상 소요와 우선순위를 함께 제시할 것 먼저 전체 실행계획과 위험 지점 3개만 제시하고 멈춰라. 계획이 승인되면 1단계만 실행하라. 각 단계가 끝날 때 통과 기준에 따른 자체 검수와 다음 단계에 넘길 확정 입력을 요약하라.

결과 체크 3문항

  • 각 단계에 눈으로 확인할 수 있는 산출물이 있는가?
  • 앞 단계의 통과 기준을 만족해야 다음 단계가 시작되는가?
  • 사람이 판단해야 할 결정과 ChatGPT가 수행할 작업이 구분됐는가?

자주 하는 실수

  • “단계별로 생각해”라고만 쓰고 단계 산출물을 정하지 않기
  • 조사와 초안 작성을 동시에 시켜 근거와 문구가 섞이게 하기
  • 모든 단계에 승인을 넣어 흐름을 지나치게 느리게 만들기
  • 계획을 받은 뒤 검토하지 않고 곧바로 전체 실행시키기

오늘의 미션

진행 중인 공공기관 제안 업무 하나를 5단계로 분해하고, 각 단계의 산출물·통과 기준·사람 확인 여부를 표로 만드세요. 가장 되돌리기 비싼 결정 앞에 승인 게이트를 하나 넣으면 완료입니다.
