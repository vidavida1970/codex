# DAY 9 — 복잡한 업무를 단계별로 나누는 법

**발행일:** 2026-07-23  
**브랜드:** 애드아이티 · ChatGPT로 성공하기

## 10초 훅

큰 업무를 한 번에 맡기면 오류도 한 번에 커진다. 단계마다 작은 완료를 만들자.

## 오늘의 학습목표

복잡한 업무를 단계별 산출물과 검토 지점으로 나눠, 앞 단계의 오류가 최종 결과까지 번지는 것을 막는다.

## 핵심 개념

작업 분해는 긴 프롬프트를 여러 문장으로 나누는 일이 아니다. 각 단계에 입력·작업·산출물·통과 기준을 붙이는 업무 설계다.

1. 정의: 목표·사용자·최종 완료 기준 확정
2. 분석: 요구사항·사실·빈칸 분리
3. 설계: 전략·구성·우선순위 결정
4. 제작: 승인된 설계로 초안 작성
5. 검수: 근거·누락·형식·리스크 점검

## 잘못된 사용법과 개선

- 잘못: “RFP를 분석하고 전략을 세워 50장 제안서를 완성해줘.”
- 개선: 요구사항표 → 쟁점·빈칸 → 전략 3안 → 승인안 장표 구성 → 심사표 검수로 나누고 단계마다 통과 기준을 둔다.

## 애드아이티 업무 적용

공공기관 홍보 제안을 RFP 요구사항표 → 평가항목별 과제 → 타깃 인사이트 → 채널 전략 → 콘텐츠 구성 → 장표 초안 → 심사표 검수로 나눈다. 각 단계가 끝날 때 확인된 사실·추론·추가 자료를 분리해 승인한다.

## 오늘의 실제 사례

Axios Local은 뉴스레터 전체를 한 번에 자동화하지 않고 구성요소별로 나눴다. 기자가 취재와 초안을 맡고, Axiomizer GPT가 제목과 ‘왜 중요한가·다음은 무엇인가’를 점검한 뒤 편집자가 판단한다. Axios는 반복 제작·형식 작업 시간을 줄여 기자와 편집자가 취재와 판단에 집중하고 1인 기자 도시 운영을 지원했다고 설명했다. 정량 성과는 공개되지 않았다.

- 원문: https://openai.com/index/axios-allison-murphy/
- 2026-07-23 공식 사례 모음: https://openai.com/index/how-news-organizations-are-using-ai/
- 국내에서 단계 분해 전후를 측정한 검증 가능한 공개 사례는 확인하지 못했다.

## 2분 따라 하기

1. 복잡한 업무 하나와 최종 결과물을 적는다.
2. 시작부터 완료까지 동사 5개로 나눈다.
3. 각 단계의 산출물과 통과 기준을 붙인다.
4. 되돌리기 비용이 큰 단계 앞에 승인 게이트를 둔다.

## 복사형 실전 프롬프트

```text
[최종 목표]
{완성할 결과물과 사용처}

[제공 자료]
{RFP·사업자료·기존 콘텐츠·통계 등}

[완료 기준]
{누가 무엇을 확인하면 완료인지}

이 업무를 4~7개의 실행 단계로 분해하라.
각 단계는 아래 필드를 포함한다.

| 단계 | 입력 자료 | 수행 작업 | 단계 산출물 | 통과 기준 | 사람 확인 |

[분해 원칙]
- 앞 단계 산출물이 다음 단계 입력이 되게 할 것
- 조사·판단·작성·검수를 한 단계에 섞지 말 것
- 근거가 부족하면 ‘추가 자료 필요’로 표시할 것
- 되돌리기 비용이 큰 결정 앞에는 승인 게이트를 둘 것
- 단계별 예상 소요와 우선순위를 함께 제시할 것

먼저 전체 실행계획과 위험 지점 3개만 제시하고 멈춰라.
계획이 승인되면 1단계만 실행하라.
각 단계가 끝날 때 통과 기준에 따른 자체 검수와
다음 단계에 넘길 확정 입력을 요약하라.
```

## 결과 체크

- 각 단계에 눈으로 확인할 수 있는 산출물이 있는가?
- 앞 단계의 통과 기준을 만족해야 다음 단계가 시작되는가?
- 사람의 판단과 ChatGPT의 수행 작업이 구분됐는가?

## 자주 하는 실수

- “단계별로 생각해”라고만 쓰고 단계 산출물을 정하지 않기
- 조사와 초안 작성을 동시에 시켜 근거와 문구를 섞기
- 모든 단계에 승인을 넣어 흐름을 지나치게 느리게 만들기
- 계획을 검토하지 않고 전체 실행시키기

## 오늘의 미션

공공기관 제안 업무 하나를 5단계로 분해하고 각 단계의 산출물·통과 기준·사람 확인 여부를 표로 만든다. 가장 되돌리기 비싼 결정 앞에 승인 게이트를 하나 넣는다.

## 최신 업데이트

2026-07-23 OpenAI는 뉴스 조직들이 데이터 탐색, 요청서 작성, 제목·문체 점검, 요약·순위화처럼 경계가 명확한 작업에 AI를 배치하고 사람이 편집 판단을 유지하는 사례를 공개했다. 제품 기능 릴리스는 2026-07-16 데스크톱 앱 개편 이후 추가 변경이 확인되지 않았다.
